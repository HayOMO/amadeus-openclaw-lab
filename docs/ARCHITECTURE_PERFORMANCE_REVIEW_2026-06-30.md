# Architecture Performance Review - 2026-06-30

Scope: keep the 50 Telegram-chat feature ideas as backlog/prototype work, then
optimize the current runtime architecture without expanding the active tool
surface.

## Backlog Boundary

- `imagebot-chat-toolbox` remains a local prototype with tests and manual notes.
- Its 50 concrete backlog actions are now grouped behind 9 evaluation bundles
  (`task`, `records`, `schedule`, `poll`, `random`, `text`, `wellness`,
  `social`, `queue`) so future review does not treat every small command as a
  separate top-level design surface.
- It is not present in `config/imagebot/settings.json` allowed plugin/tool
  lists.
- It is not generated by `scripts/BUILD_IMAGEBOT_CONFIG.mjs`.
- `docs/bot_feature_scouting/telegram_chat_new_features_2026-06-30.md` tracks
  the 50 items as `todo_candidate`, not shipped runtime behavior.

## Optimizations

- Tool surface consolidation
  - Active `allowedTools` was reduced from the previously observed 71 to 61 by
    replacing read-only split tools with action-based aggregate tools.
  - New public aggregate tools:
    `generated_gallery`, `media_artifact`, `artifact`, `image_skill`,
    `knowledge`, and `zhihu`.
  - Legacy split tools remain registered internally for compatibility tests, but
    are no longer part of the public allowed tool list or manifest contract.
  - Operator-only write/mutation actions stay separate: `generated_gallery_resend`,
    `image_skill_save_reference`, `image_skill_note_preference`, and
    `knowledge_ingest` were not folded into public read aggregators.
  - `web_watch_*` remains split because list is a read action while add/check/delete
    are operator/side-effect actions; merging it safely needs action-level
    permission support first.
- `tool_manual_search`
  - Removed synchronous full-file reads during module import.
  - Caches parsed manual sections using manual file size/mtime signatures.
  - Keeps focus enum coverage tested against current manual frontmatter ids.
- `imagebot-agent-ops`
  - Runs persona, active mode, and learned-skill prompt context collection in
    parallel.
  - Caches parsed JSON state by file size/mtime and invalidates after atomic
    writes.
- `imagebot-creative-ops`
  - Caches parsed append-only JSONL logs for script history and image feedback.
  - Invalidates cached records after each append.
- `imagebot-memory-search`
  - Caches the known-user map from the Telegram window store by size/mtime.
  - Returns a fresh `Map` per call to avoid accidental caller mutation.
